- [ ] В корне репозитория `mcu` находится `.gitignore` файл;
- [ ] В `.gitignore` добавлена папка сборки проектов `build/`;
- [ ] В `.gitignore` добавлена папка локальных настроек `.vscode/`.
- [ ] Сделать `commit` и `push` файла `.gitignore` в репозиторий `mcu`

1. Создать в корне вашего репозитория `mcu` пустой файл с именем:

```
.gitignore
```

> [!ATTENTION] Обратите внимание!
>
>  Имя файла должно быть в точности `.gitignore`, т. к. это зарезервированное имя файла для системы git. В этом файле хранятся шаблоны имен файлов и папок, которые git должен игнорировать. Такие файлы не будут отображаться в `git status` и не будут попадать в `git add`.

2. Открыть файл в VSCode и добавить в `.gitignore` папку для сборки прошивок:

```
build/
```

> [!INFO] Примечание
> 
> В дальнейшем, в каждом проекте мы будем создавать папку `build` и собирать прошивки именно в ней. При сборке, в этой папке будет множество временных файлов, которые версионироваться **не должны**.

> [!INFO] Справка
> 
> `.gitignore` поддерживает шаблоны, поэтому одной строчки `build/` достаточно для того, чтобы исключить все папки `build` из всех проектов нашего репозитория.

3. Добавить в `.gitignore` папку с локальными настройками VSCode, которую приложение VSCode может создать автоматически:

```
.vscode/
```

> [!INFO] Примечание
> 
> Для разных проектов и языков (C, C++, Python, JS и т. д.) есть набор типовых шаблонов для исключений, в которые добавляются свойственные языку и системе сборки кэши, папки сборки, временные файлы и т.д. Зачастую, используются стандартные шаблоны, которые может сгенерировать, например, сайт github. В нашем курсе используется фундаментальная система сборки через `cmake` и `make` без лишних инструментов, поэтому достаточно `.gitignore` всего двух строчек.

4. Добавить файл `gitignore` в index, сделать `commit`, `push`:

``` bash
git add .gitignore
git commit -m "Added .gitignore file"
git push
```
